🔐 1. Authentication
Verify login works and the operator lands on the dashboard.
Login as operator
  1. Go to /login
  2. Enter email + password
  3. Click Sign In
  4. ✅ Should land on Dashboard — not stuck on login
👤 2. Client Setup
Admin can add a client and configure their portal in under 10 minutes.
Add a new client
  1. Go to /clients → click Add Client
  2. Fill in name, email, service type
  3. Click Save
  4. ✅ Client appears in the list
Set up portal links (intake, contract, payment, access)
  1. Open the client → Portal tab
  2. Add an intake form URL, contract URL, payment URL
  3. Click Save
  4. ✅ Links are saved and visible
Set up onboarding stages
  1. Open the client → Portal tab → Onboarding Stages section
  2. Add stages: e.g. Contract Signed, Kickoff Call, Access Granted
  3. Mark one stage as complete
  4. ✅ Stages save and show correct status
Generate magic link for client portal
  1. Open the client → Portal tab
  2. Click Generate Magic Link
  3. Copy the link
  4. ✅ Link is generated and copyable
  5. Open link in an incognito window → ✅ Portal loads without login
📦 3. Deliveries
Operator can log work and the client sees it in the portal.
Log a delivery
  1. Open client → Deliveries tab
  2. Click Log Delivery
  3. Fill in title, category, status
  4. ✅ Delivery appears in the list
Client sees delivery in portal (Work Done)
  1. Open the portal link in incognito
  2. Go to Work Done section
  3. ✅ Delivery logged above appears here
Client approves a delivery
  1. Log a delivery with status Pending Approval
  2. Open portal in incognito → Home section
  3. ✅ Delivery shows in Awaiting You stat card
  4. Click Approve
  5. ✅ Counter drops to 0, delivery moves to approved
💬 4. Client Requests (Brain Dump)
Client can submit a request from the portal and operator can manage it.
Client submits a request from portal
  1. Open portal in incognito
  2. Click Request Something button
  3. Fill in title + description → Submit
  4. ✅ Success confirmation shown
  5. Go to Ask Your Team section → ✅ Request appears with status Pending
Operator sees and updates the request
  1. Go to operator app → client → Requests tab
  2. ✅ Request submitted above appears here
  3. Change status to In Review
  4. Open portal in incognito → ✅ Status reflects the change
📊 5. Scope & Plan
Scope usage is tracked and shown to the client.
Scope tracker shows correct usage
  1. Open client → Scope tab
  2. ✅ Allocated hours/budget and used amount shown
  3. ✅ Progress bar fills correctly
  4. Open portal → Home → Plan Usage card → ✅ Matches
Scope alert shows at 80%+ usage
  1. Set a client's scope allocation low enough that usage is over 80%
  2. ✅ Amber warning banner appears in Scope tab
  3. ✅ Dashboard shows the client in the alerts section
🎨 6. Portal Experience
Portal is branded, mobile-friendly, and shows the right info.
Portal loads without login via magic link
  1. Open magic link in incognito / private browser
  2. ✅ Portal loads — no login screen
  3. ✅ Client name and branding shown in header
  4. ✅ Stat cards (Deliveries, Plan Usage, Awaiting You) visible
Portal is usable on mobile
  1. Open portal link on your phone (or DevTools → iPhone 14)
  2. ✅ No horizontal scrolling
  3. ✅ Buttons are large enough to tap
  4. ✅ Navigation works
  5. ✅ Request form opens and submits on mobile
Invalid or expired portal link shows a clear error
  1. Open /portal/this-is-not-real in browser
  2. ✅ Error page shown — not a blank screen
  3. ✅ Message says the link isn't valid
  4. ✅ Try again button visible
⚙️ 7. Settings & Pick Lists
Operator can manage pick lists for deliverables and uplifts.
Pick lists can be managed in Settings
  1. Go to /settings
  2. Find the Pick Lists section
  3. Add a new deliverable item
  4. ✅ Item saves and appears in the list
  5. Delete it → ✅ Item removed
📝 8. Proposals
Create and send a proposal
  1. Go to /proposals/new
  2. Fill in client, title, add a content block
  3. Click Save
  4. ✅ Proposal saved and appears in proposals list
  5. Open proposal → copy portal link → open in incognito → ✅ Proposal visible to client
